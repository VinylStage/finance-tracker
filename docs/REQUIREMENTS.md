# 기능 요구사항

## 실제 구현 기준으로 역산

### 거래 (transactions) CRUD 및 조회
- 거래 내역에 대한 전체 CRUD 작업 가능 (POST, GET, PUT, DELETE)
- 기간별 및 카테고리별 조회 지원 (거래 목록 조회, 카테고리별 지출 분석 등)
- 가맹점명 입력 시 자동 카테고리 추천 기능 제공

### 할부 (installments)와 리볼빙 (revolving) 관리
- 일반 거래와 구분하여 별도 테이블 관리
  - `installments`: 분할 결제 정보 저장 (총액, 개월수, 월할부금 등)
  - `revolving_history`: 신용카드 회계 기록 저장 (이월잔액, 신규충전, 지불금액 등)
- 할부 및 리볼빙 관련 정보는 전용 엔드포인트 제공
- 대시보드 집계 시 할부 및 리볼빙 분석 포함

### 예산과 실적 비교
- 카테고리별 월간 예산 설정 및 관리 가능
- 실제 지출 내역과 예산 비교 분석 (대시보드에서 예산 초과 경고)

### 대시보드 집계 기능
- 수입/지출/할부/리볼빙/카테고리별 분석 정보 제공
- 일별, 주별, 월별, 년별 기간 비교 분석 가능
- 최근 30일/12주/12개월 트렌드 그래프 포함
- 상위 가맹점 목록 조회

### 부채 (debts) 및 저축 (savings) 관리
- 부채 정보 및 이자 기록 저장 및 조회 가능
- 저축 상품 등록, 수정, 삭제 가능
- 적금 상품 만기 처리 기능 지원

### 데이터 내보내기 (export) 기능
- CSV/JSON 형식으로 거래내역 내보내기
- 시스템 내에서 내보내기 API 제공 (`/api/export`)
- 특정 기간의 데이터 내보내기 가능

### 추가 확인된 기능들
- 결제수단 및 카테고리 관리 (CRUD)
- 애플리케이션 설정 관리 (기초 자산, 월간 수입 등)
- 현금흐름 분석 API 제공
