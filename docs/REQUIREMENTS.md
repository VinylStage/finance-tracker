# 기능 요구사항

## 배경 및 목적

Google Sheets / Excel 기반 가계부(v1~v5)를 로컬 웹앱으로 전환한다.
스프레드시트 대비 해결해야 할 핵심 문제:
1. 일/주/월/연 단위 실시간 현금흐름·지출 그래프
2. 결제수단 목록을 코드 수정 없이 자유롭게 추가·관리
3. 카테고리 자동제안 + 수동 선택 병행

1인 가구, 로컬 전용, 인증 불필요.

## 기능 요구사항 (FR)

### FR-1. 거래 입력
필드: 날짜, 대분류(유형), 소분류(카테고리), 금액, 결제수단, 결제방식, 가맹점/내용, 메모.
결제방식 = '할부' → 할부 등록 화면으로 유도.
결제방식 = '리볼빙' → 리볼빙 원장과 연동.
최근 사용 카테고리·결제수단 우선 노출.

### FR-2. 카테고리 자동제안
가맹점명 입력 시: 정확 일치 → 가장 최근 카테고리; 부분 일치 → 빈도 최상위; 매칭 없으면 빈 값.
미리 선택된 상태로 표시하되 드롭다운으로 수동 변경 가능.

### FR-3. 결제수단 관리
설정 화면에서 카드/결제수단 추가 (이름, 유형: 신용/체크/이체/현금성).
삭제 없음 — 비활성화(is_active=0)로 처리.
초기 시드: 하나/삼성/현대/신한/롯데/농협카드, 현금성결제, 현금, 자동이체, 계좌이체.

### FR-4. 할부 관리
구매 등록: 상호, 총액, 개월수, 카드사, 시작청구월.
월상환액 자동 계산 (수수료 입력 시 반영).
대시보드에 이번달 청구 예정 할부 목록 자동 집계.
남은 회차 0 도달 시 '완료' 상태 자동 전환.

### FR-5. 리볼빙 관리
월별 이월잔액, 신규청구, 결제액, 이자 기록.
이자는 '대출이자' 카테고리로 자동 분류 (소비 통계와 분리).

### FR-6. 부채 현황
마이너스통장 등 잔액 수동 입력 + 이자율 입력 시 월 이자 자동 계산.
카드 미결제 잔액, 할부 잔여, 리볼빙 잔여 자동 집계.
총부채 = 위 항목 합계.

### FR-7. 적금·저축성보험
상품 등록: 상품명, 월납입액, 시작일, 만기일, 예상 만기수령액.
납입 시 '저축' 카테고리로 자동 기록.
만기 처리: 원금 회수(저축 마이너스) + 이자만 '수입'으로 분리 기록 → 이중계산 방지.

### FR-8. 대시보드
- 순자산 = 가용현금 + 투자자산 + 누적저축 - 총부채
- 이번달 결제예정 총액 (할부+리볼빙+고정지출+부채상환)
- 예산 대비 실적 (카테고리별, 이번달 기준)
- 예상잔액 시뮬레이터: 월 수입/고정비/부채상환/저축 가정값 → N개월 후 잔액

### FR-9. 현금흐름·지출 그래프
- 시점 전환 탭: 일별 / 주별 / 월별 / 연도별
- 현금흐름: 선형 그래프 (잔액 추이)
- 지출: 카테고리별 누적 막대 또는 선형 (사용자 전환 가능)
- 거래 입력 즉시 그래프 갱신 (새로고침 불필요)

## 비기능 요구사항
- 실행 환경: 로컬 PC (Node.js, `npm start` 한 줄)
- 데이터 저장: SQLite 단일 파일, 로컬 보관
- 백업: DB 파일 복사만으로 완료; 내보내기 버튼(CSV/JSON) 제공
- 반응형: PC 브라우저 우선, 모바일 입력 가능한 최소 대응
- 성능: 3년치(약 5,000건) 기준 모든 조회 1초 이내
